// 01 — Definizione

Che cos'è TLS / SSL?

TLS (Transport Layer Security) e SSL (Secure Sockets Layer) sono protocolli crittografici che creano un canale cifrato tra due dispositivi in rete, proteggendo ogni byte trasmesso da intercettazioni.

📜 SSL — Il Predecessore

Creato da Netscape nel 1994 per proteggere le prime transazioni bancarie. Sostituito da TLS per vulnerabilità strutturali.

SSL 2.0 ✗SSL 3.0 ✗

🔐 TLS — L'Erede

Standardizzato da IETF nel 1999. Versione attuale: TLS 1.3 (2018) — più veloce, più sicuro, forward secrecy obbligatoria.

TLS 1.2TLS 1.3 ✓

🌐 HTTPS = HTTP + TLS

Il lucchetto 🔒 nel browser = TLS attivo. Google penalizza HTTP dal 2018. Obbligatorio per qualsiasi sito moderno.

Porta 443vs :80 HTTP

🏗️ Dove opera nello stack

TLS si inserisce tra il livello TCP e il livello applicativo. Avvolge qualsiasi protocollo: HTTP, SMTP, FTP, MQTT, LDAPS…

App LayerTLS ← quiTCPIP

🔑 Crittografia Ibrida

Asimmetrica (RSA / ECDHE) per scambiare chiavi → Simmetrica (AES-256) per cifrare a velocità hardware. Il meglio di entrambi.

RSA-2048+AES-256-GCMECDHE
// 02 — Motivazione

Perché utilizzarlo?

Senza TLS i dati viaggiano in chiaro: chiunque sulla rete può intercettarli, leggerli e modificarli.

🔒 Riservatezza

I dati sono cifrati end-to-end. Anche intercettando il traffico, il testo è incomprensibile senza la chiave segreta di sessione.

AES-256-GCMChaCha20

✅ Integrità

Ogni messaggio include un codice di autenticazione (MAC). Se qualcuno modifica un solo byte in transito, il destinatario lo rileva e scarta.

HMAC-SHA256AEAD

🪪 Autenticazione

I certificati digitali X.509 firmati da una CA provano l'identità del server. Nessun impostore può fingere di essere google.com.

X.509PKI / CAECDSA

⚡ Senza TLS — Rischi reali

  • Password e carte di credito rubate in chiaro
  • Attacchi Man-in-the-Middle: l'hacker legge tutto
  • Injection di pubblicità o malware nelle pagine
  • Intercettazione da ISP, governi, Wi-Fi pubblici

🏆 Con TLS — Garanzie

  • Dati cifrati: illeggibili anche se intercettati
  • Google ranking migliore: HTTPS è fattore SEO
  • Utenti fidati: il lucchetto trasmette sicurezza
  • Conformità legale: GDPR, PCI-DSS, HIPAA

🐢 TLS 1.2 — Vecchio standard

  • 2 round-trip per completare l'handshake
  • Algoritmi deboli: RSA key exchange, RC4, MD5
  • Nessuna forward secrecy garantita di default
  • Cipher deboli opzionali → rischio misconfiguration
  • Ancora valido, ma deprecato in favore di 1.3
2 RTTRSA kexWeak ciphers

🚀 TLS 1.3 — Standard moderno (2018)

  • 1 round-trip — handshake 50% più veloce
  • Rimossi tutti gli algoritmi deboli (RC4, MD5, RSA…)
  • Forward Secrecy obbligatoria via ECDHE
  • 0-RTT per connessioni ripetute
  • Header handshake cifrati: meno metadati esposti
1 RTTECDHE force0-RTTFS

// 04 — Applicazioni nel Mondo Reale

Dove viene usato?

TLS protegge praticamente ogni byte su Internet — dal browser al frigorifero smart.

🌐HTTPS / WEB

Ogni sito con 🔒. Chrome blocca HTTP dal 2018. Oltre il 95% del traffico usa TLS.

📧EMAIL

SMTPS (:465), IMAPS (:993), POP3S. Gmail, Outlook: TLS obbligatorio.

💳PAGAMENTI

PayPal, carte di credito, home banking. PCI-DSS richiede TLS 1.2+.

💬MESSAGGISTICA

WhatsApp, Telegram, Signal. TLS protegge la connessione ai server.

☁️CLOUD / API

AWS, Azure, GCP. Ogni REST/GraphQL richiede HTTPS con TLS valido.

🎮GAMING

Login, acquisti, chat. Steam, PlayStation, Xbox Live: TLS su tutto.

📱APP MOBILE

iOS ATS e Android Network Security Config impongono TLS di default.

🏥SANITÀ

Cartelle cliniche, telemedicina. GDPR e HIPAA lo rendono obbligatorio.

🏭IoT / INDUSTRIA

Sensori, PLC, SCADA. MQTT over TLS protegge reti industriali.

// 05 — Analisi Critica

Vantaggi & Svantaggi

✅ Vantaggi

  • Sicurezza militare — AES-256, forward secrecy, protezione MITM
  • Standard universale — Tutti i browser, OS e dispositivi
  • TLS 1.3 è velocissimo — 1 RTT, 0-RTT per sessioni ripetute
  • Certificati gratuiti — Let's Encrypt + rinnovo automatico
  • SEO & fiducia — Google premia HTTPS, lucchetto rassicura
  • Conformità legale — GDPR, PCI-DSS, HIPAA, NIS2

✕ Svantaggi

  • Overhead CPU — Crittografia consuma risorse (minimo su hardware moderno)
  • Latenza iniziale — L'handshake aggiunge ritardo alla prima connessione
  • Gestione certificati — Scadono ogni 90 giorni; richiedono rinnovo
  • Config errata = vulnerabile — TLS 1.0/1.1 azzerano la sicurezza
  • Falsa sicurezza — Protegge il transito, non i dati archiviati
  • Debugging difficile — Traffico cifrato complica l'analisi

⚠️ Versioni da evitare

SSL 2.0/3.0, TLS 1.0/1.1: deprecati. Usare TLS 1.2 come minimo, TLS 1.3 idealmente.

SSL ✗TLS 1.0 ✗TLS 1.1 ✗

🏆 Verdetto Finale

Pro >> Contro. TLS è imprescindibile. Non è una scelta, è una necessità tecnica e legale per ogni app moderna.
